 PERBAIKAN KECIL YANG DITEMUKAN SAAT TES LIVE (bukan endpoint baru)
==============================================================================

 1. [BUG] PATCH /orders/{id}/status balas 500 (padahal aksinya SUKSES)
    Status di DB berubah & push C4 terkirim, tapi response-nya 500 karena
    OrderOut.model_validate(order) lazy-load order_items/updated_at setelah
    commit (MissingGreenlet, async SQLAlchemy).
    Fix: di order_service.update_order_status, setelah commit re-query order
    by id dengan options(selectinload(Order.order_items),
    selectinload(Order.invoice)) sebelum return — sama polanya dengan yang
    kamu pakai di get_customer_latest_order.

 2. [BUG] Cancel order tidak mengembalikan stok bahan
    POST /orders sudah MENGURANGI stok bahan (bagus), tapi
    POST /orders/{id}/cancel hanya set status cancelled — bahan yang sudah
    dipotong tidak dikembalikan. Order batal/expired = stok bocor.
    Fix: di cancel_order_by_customer, kembalikan stok per resep
    (stok_tersedia += jumlah_dibutuhkan * jumlah item).

 3. [CONFIG] Set env CHATBOT_URL di backend
    Default config chatbot_url = http://localhost:8001 (itu port backend
    sendiri!). Supaya push C4 sampai, set di .env backend:
        CHATBOT_URL=http://localhost:8000
    (atau alamat Chatbot Service di deployment).

 4. [NICE-TO-HAVE] Endpoint Owner untuk set handles_takeover
    Kolom users.handles_takeover sudah ada dan GET /admin/takeover-handlers
    sudah jalan. Tapi belum ada endpoint buat Owner mengubahnya dari Admin
    Site (sekarang harus lewat SQL manual):
        PATCH /users/{user_id}/takeover-handler   (require_owner, JWT)
          Request : { "handles_takeover": true }
    Ini urusan Admin Site, bukan chatbot — chatbot sudah jalan tanpa ini.

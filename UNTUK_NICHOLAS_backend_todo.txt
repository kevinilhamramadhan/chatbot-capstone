             setelah await db.commit() objek `order` di-expire; waktu route
             memanggil OrderOut.model_validate(order), pydantic menyentuh

    Fix    : setelah commit, JANGAN return objek lama — re-query dulu:

               result = await db.execute(
                   select(Order)

                   .options(selectinload(Order.order_items),
                            selectinload(Order.invoice))

               return result.scalars().first()

 2. [BUG] Cancel order tidak mengembalikan stok bahan
    Gejala : POST /orders sudah MENGURANGI stok bahan (bagus), tapi
             POST /orders/{id}/cancel cuma set status=cancelled — bahan yang
             sudah dipotong tidak dikembalikan. Order batal = stok bocor,
             lama-lama stok sistem < stok fisik.
    Fix    : di cancel_order_by_customer() (app/services/order_service.py),
             sebelum/bersamaan set cancelled:
             a. Muat order.order_items beserta product.recipes + stock_item
                (selectinload berantai, seperti di create_new_order).
             b. Untuk tiap item x tiap resep produknya:
                  kembalikan = recipe.jumlah_dibutuhkan * item.jumlah
                  UPDATE stock_items
                     SET stok_tersedia = stok_tersedia + kembalikan,
                         version = version + 1
                   WHERE id = recipe.stock_item_id
             c. Commit bareng perubahan status (satu transaksi).
    SELESAI JIKA: catat stok bahan -> buat order (stok turun) -> cancel order
             -> stok kembali persis ke angka semula.

 4. [ENDPOINT BARU — SATU-SATUNYA YANG MASIH DITUNGGU CHATBOT]
    Guna: fitur chatbot untuk Owner — tanya "laporan keuangan" / "produk
    terlaris" via WA -> bot balas angka REAL. Sisi chatbot SUDAH SELESAI &
    sudah dites (dengan endpoint ini di-mock); begitu kamu deploy, fitur
    langsung hidup tanpa perubahan apa pun di chatbot.

      GET /reports/summary?start_date=YYYY-MM-DD&end_date=YYYY-MM-DD
      (Depends(require_service_key) — sama seperti endpoint chatbot lain)

      Response 200:
      {
        "revenue": 12500000.0,
        "expenses": 7300000.0,
        "order_count": 34,
        "avg_order_value": 185000.0,
        "top_products": [
          {"nama_produk": "Brownies Coklat", "qty": 12, "revenue": 600000.0}
        ]
      }

    Cara hitung tiap field (nama kolom sudah kucocokkan dengan modelmu):
      revenue         = SUM(payments.jumlah_bayar)
                        WHERE payment_status = 'Success'
                          AND created_at antara start_date..end_date
      expenses        = SUM(expenses.jumlah)
                        WHERE tanggal antara start_date..end_date
                        (logika /expenses/summary/dashboard kamu sudah ada —
                         tinggal dipakai ulang, bedanya auth service-key)
      order_count     = COUNT(orders) WHERE created_at di periode
                          AND status != 'cancelled'
      avg_order_value = AVG(orders.total_harga_pesanan), filter sama
      top_products    = JOIN order_items -> orders (filter periode &
                        non-cancelled) GROUP BY product_id:
                          qty     = SUM(order_items.jumlah)
                          revenue = SUM(order_items.subtotal)
                        ORDER BY qty DESC LIMIT 5, ambil products.nama_produk
    Periode kosong -> nilai 0 / list kosong (jangan 404).
    SELESAI JIKA: curl -H "X-Service-Key: <key>"
      "http://localhost:8001/reports/summary?start_date=2026-06-01&end_date=2026-06-30"
      balas angka yang cocok dengan isi DB. (Chatbot memanggil dengan rentang
      bulan berjalan.)
